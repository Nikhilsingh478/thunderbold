---

# 🧠 CORE BUSINESS LOGIC (VERY IMPORTANT)

A user can ONLY review a product IF:

1. The user has placed an order containing that product
2. The order status is **"delivered"**

👉 If BOTH conditions are satisfied → user can review
👉 Otherwise → user CANNOT review

---

# 🧱 DATABASE DESIGN

Create a new collection: `reviews`

Schema:

```js
{
  _id: ObjectId,
  userId: string (user email or uid),
  productId: string,
  orderId: string, // to ensure authenticity
  rating: number (1–5),
  comment: string,
  isDeleted: boolean (default: false),
  createdAt: Date,
  updatedAt: Date
}
```

---

# 🔐 BACKEND LOGIC

## 1. Create Review

Endpoint:
POST /api/reviews

Validation:

* Verify user is authenticated
* Verify:

  * user has an order with this product
  * AND order.status === "delivered"
* Prevent duplicate reviews:

  * one user → one review per product

If review exists → return error OR allow update instead

---

## 2. Update Review

PUT /api/reviews/:id

Rules:

* Only review owner can update
* Allow editing:

  * rating
  * comment

---

## 3. Delete Review (User)

DELETE /api/reviews/:id

* Soft delete only:

```js
isDeleted: true
```

---

## 4. Get Reviews (Public - Product Page)

GET /api/reviews?productId=xxx

* Return only:

```js
isDeleted: false
```

* Sort by latest

---

## 5. Admin Delete Review

Endpoint:
DELETE /api/admin/reviews/:id

* Admin only
* Soft delete
