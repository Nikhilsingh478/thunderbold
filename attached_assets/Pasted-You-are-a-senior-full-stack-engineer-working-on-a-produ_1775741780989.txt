# 🚨 PRIORITY 1: BULLETPROOF ORDER IDEMPOTENCY (CRITICAL)

## Problem:

Current idempotency (clientOrderId) is NOT fully safe under race conditions.

## Task:

Enforce true uniqueness at database level.

### Steps:

1. Create a UNIQUE index on orders collection:

```js
db.orders.createIndex({ clientOrderId: 1 }, { unique: true });
```

2. Update `/api/orders/create.js`:

* Wrap order creation in try/catch
* If duplicate key error occurs (E11000):
  → fetch and return existing order instead of failing

Example:

```js
try {
  await db.orders.insertOne(orderData);
} catch (error) {
  if (error.code === 11000) {
    const existing = await db.orders.findOne({ clientOrderId });
    return existing;
  }
  throw error;
}
```

# 🚨 PRIORITY 2: ROLE-BASED ADMIN AUTH (SECURITY)

## Problem:

Admin is currently determined by:

```js
user.email === ADMIN_EMAIL
```

## Task:

Move to database-driven roles.

### Steps:

1. Update `users` collection schema:

```json
{
  "email": "string",
  "role": "user | admin"
}
```

2. On user creation (`/api/users`):

* Default role = "user"

3. Create helper:

```js
async function isAdmin(email) {
  const user = await db.users.findOne({ email });
  return user?.role === "admin";
}
```

4. Replace ALL admin checks:

* Use DB role instead of env email

5. (Optional fallback)

* If no role found, fallback to ADMIN_EMAIL
